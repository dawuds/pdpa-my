# Anatomy of a GDPR audit pack

> **Tier 4 / Lesson 1 — 45 minutes.** Parallel to PDPA T4 Lesson 1. What a GDPR audit pack contains.

## What "GDPR audit pack" means

GDPR audits typically take one of three forms:

| Form | Driver | Output |
|---|---|---|
| **Voluntary controller / processor audit** | Programme assurance; readiness assessment | Internal report; remediation plan |
| **Article 28 controller-of-processor audit** | Controller's audit right under DPA | Report shared with controller; processor remediation |
| **Compliance assessment under SA investigation** | Supervisory authority requested audit | Report shared with SA (and counsel) |

The audit pack structure is largely the same across forms; the audience and discovery posture differ.

## Core artefacts of a GDPR audit pack

| Artefact | Purpose |
|---|---|
| **Engagement letter** | Scope, objective, methodology, deliverable, fees |
| **Work programme** | Article-anchored procedures (Articles 5-50 typical coverage) |
| **Information request list** | The GDPR equivalent of PBC — usually 60-120 items |
| **Rating and conclusion methodology** | Defining controller / processor obligations + severity scale |
| **Citation framework** | Article + EDPB Guideline version + national SA guidance pinning |
| **Finding template** | 5 Cs equivalent + Article-based citation |
| **Audit report template** | Executive summary + scope + findings + conclusions + appendices |
| **Reference set** | Consolidated GDPR text; EDPB Guidelines (current versions); national SA guidance; relevant CJEU decisions |

## The standards underpinning GDPR audit

Three reference standards inform GDPR audit work:

| Standard | Use |
|---|---|
| **ISO/IEC 27701:2019** | Privacy Information Management System — the de facto controller / processor assurance backbone; provides Annex A controls aligned with GDPR |
| **ISO/IEC 27001:2022** | ISMS — underpins Article 32 security; almost universal in regulated industries |
| **AICPA Trust Services Criteria + SOC 2** | US-origin but widely used internationally; the Privacy criterion overlaps Article 5 + Article 32 |
| **EuroPriseseal / IPC GmbH** | Privacy seals; specific to particular processing activities |

For a typical EU controller engaging an audit, the assurance backbone is ISO 27701 + ISO 27001. The audit programme tests Article-specific obligations against the ISO 27701 controls.

## The reference set

A defensible GDPR audit references:

| Source | Where |
|---|---|
| Consolidated Regulation text | EUR-Lex; latest consolidated version including any subsequent amendments |
| EDPB Guidelines (issued versions, not drafts) | edpb.europa.eu |
| EDPB Opinions and Article 65 decisions | edpb.europa.eu |
| National SA guidance (e.g., ICO, CNIL, DSK, Garante, AEPD) | Per Member State |
| CJEU case law on GDPR | curia.europa.eu |
| Implementing acts (2021 SCCs etc.) | EUR-Lex Implementing Decisions |
| Adequacy decisions | Commission adequacy list |

The reference set should be **version-pinned at engagement start**. EDPB Guidelines update; CJEU decisions issue; SA blacklists revise. A finding citing an EDPB Guideline version that has since been superseded is partially defensible at best.

## Difference vs PDPA audit pack

The structural pattern is the same. Operational differences:

| Element | PDPA | GDPR |
|---|---|---|
| Methodology document | Single (Rating and Conclusion Methodology v2.3+) | Often split across multiple SA / professional body documents |
| Citation discipline | PCP-vs-issued (very specific) | EDPB version pinning + CJEU update check |
| Reference repository | 45 source PDFs locally | Reliance on web-sourced EUR-Lex / EDPB site |
| BM authoritative text | Yes — issued April 2026 guidelines BM-only | n/a — Regulation has all 24 EU language versions |
| Sectoral CoP | 8 registered + mandatory for Class | Article 40 voluntary; limited adoption |

## Common gaps in GDPR audit pack assembly

The patterns the audit team catches:

1. **Reference set not version-pinned at engagement start** — auditors work with mixed EDPB Guideline versions
2. **Member State variation not scoped** — auditor uses one SA's guidance for a multi-Member-State client
3. **CJEU updates not factored** — recent decisions superseding EDPB positions missed
4. **ISO 27701 controls not mapped to Articles** — the assurance framework exists but doesn't drive the testing
5. **Article 27 representative scope ignored** — non-EU controllers in scope under Article 3(2) audited without Article 27 representative check

## PDPA parallel

PDPA audit packs use the same architecture with stricter / more codified discipline (single methodology document; locally-stored reference PDFs; mandatory sectoral CoPs).
