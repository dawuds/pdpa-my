## 2. Purpose and Scope
The purpose of this standard is to establish a uniform framework for classifying and handling [Organization Name]'s data to ensure it receives an appropriate level of protection.
**Scope:** Applies to all physical and electronic data created, processed, or stored by [Organization Name] and its third-party processors.

## 3. Data Classification Levels
All information must be assigned one of the following four classifications:

| Level | Name | Definition | Examples |
| :--- | :--- | :--- | :--- |
| **C4** | **Restricted / Secret** | Unauthorized disclosure would cause catastrophic financial, legal, or reputational damage. | Mergers & acquisitions data, cryptographic private keys, unpatched critical vulnerabilities. |
| **C3** | **Confidential** | Unauthorized disclosure would cause significant damage or violate privacy laws. | Customer Personal Data (NRIC, Financials), employee HR files, source code. |
| **C2** | **Internal Use** | Unauthorized disclosure would cause minor operational disruption or embarrassment. | Internal memos, organizational charts, standard operating procedures. |
| **C1** | **Public** | Information explicitly approved for public release. | Press releases, public website content, published annual reports. |

## 4. Handling Requirements by Classification

| Control | Restricted (C4) | Confidential (C3) | Internal Use (C2) | Public (C1) |
| :--- | :--- | :--- | :--- | :--- |
| **Encryption (Rest)** | AES-256 | AES-256 | Optional | No |
| **Encryption (Transit)** | TLS 1.3 / VPN | TLS 1.2+ | TLS 1.2+ | Standard HTTPS |
| **Access Control** | Explicit Need-to-Know + MFA | Role-Based Access (RBAC) | All Employees | Unrestricted |
| **Email Transmission** | Secure portal / Encrypted attachment | Internal only (External requires encryption) | Allowed | Allowed |
| **Cloud Storage** | Prohibited (On-premise only) | Approved Enterprise Cloud only | Approved Enterprise Cloud | Public Cloud allowed |

## 5. Labeling Requirements
All C3 and C4 documents must be explicitly labeled in the header or footer (e.g., "CLASSIFICATION: CONFIDENTIAL"). Emails containing C3/C4 data must include the classification in the subject line.

## 6. Roles & Responsibilities
*   **Data Owners:** Assign the initial classification level to data generated by their department.
*   **Data Custodians (IT):** Implement the technical controls required for each classification level.
*   **Users:** Handle data in accordance with its assigned classification.
